FXDiagram

A framework for diagrams using JavaFX

Download this project as a .zip file Download this project as a tar.gz file

Welcome to FXDiagram.

FXDiagram is a framework for creating diagram tools based on JavaFX. By adding a thin layer of abstractions only, the developer can still leverage the full power of JavaFX. FXDiagram provides additional library classes to selectively add higher-level functionality trimmed for best user experience.

Selected features:

  • Modern graphics
  • Suggestive metaphors
  • Optimized for usability
  • Multi-touch gestures
  • Smooth transitions/animations
  • Animated auto-layout (based on KIELER and Graphviz)
  • Animated undo/redo
  • JSON persistence
  • SVG export
  • Xtext/Eclipse integration
  • Graphical context menu
  • ...and much more

FXDiagram is almost entirely written in Xtend, which allows for very concise and readable code while still being fully compatible with Java.

FXDiagram runs with plain Java applications but also integrates well with Eclipse. Adding a diagram view for an Xtext language is easy. Have a look at the examples below.

You can find more information on my blog.

Requirements

To run or develop FXDiagram you need

  • A recent Java 8 SDK u40. It must be u40 or greater to run FXDiagram without glitches,
  • Graphviz for auto-layout.

Running The Examples

FXDiagram works excellent in plain JavaFX applications. In the development environment, choose Run > Run configurations > Java Application > FXDiagram Example to run the demo. Instead of downloading and installing an entire Eclipse IDE, download fxdiagram-jars.zip, unzip it and run the contained shell script.

For Eclipse, FXDiagram ships with example diagrams for Xtext's sample languages domainmodel and fowlerdsl, as well as the generic class diagram browser for Java, Xtend and Xbase-languages. There is also a graphical tool for the analysis of Eclipse plug-in dependencies.

To give them a try, choose Run > Run configurations > Java Application > FXDiagram in Eclipse from the development environment. Alternatively, you can install the pre-compiled examples as plug-ins into an Eclipse Luna from the update site http://dl.bintray.com/jankoehnlein/generic/p2/updates.

The diagram view will automatically be opened and populated once you choose the appropriate action from the context menu:

  • Show in FXDiagram on a statemachine, a state or a transition in the fowlerdsl editor.
  • Show in FXDiagram on an entity or package in the dmodel editor.
  • Show in Class Diagram on a Java/Xtend/Xbase type in the respective editor.
  • Show in Plug-in Diagram on a plug-in project in the navigator to explore the dependencies of Eclipse plug-ins.
Most nodes can be explored using rapid buttons. Watch out for tooltips.

Development

FXDiagram is currently being developed in Eclipse. The best way to set up the development environment is to use Eclipse Oomph. Once Oomph is started

  • Switch to advanced mode (the tools icon in the top-left)
  • Choose Eclipse standard (discontinued after Luna) and enter the path to the Java 8 SDK u40 home directory in the lower part.
  • On the next page, select github.com > FXDiagram in the tree. Then double click on the new node for FXDiagram.
  • On the next page, choose the installation directories, GitHub access types and target platform. Also enter the path to the Java 8 SDK u40.
Once you click Finish, Oomph will automatically compose the IDE, start it, populate the target platform, clone the git repositories and configure the workspace. This may take a couple of minutes.
