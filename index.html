FXDiagram

A framework for diagrams using JavaFX

Download this project as a .zip file Download this project as a tar.gz file

Welcome to FXDiagram.

FXDiagram is a framework for creating diagram tools based on JavaFX. By adding a thin layer of abstractions only, the programmer can still use the full power of JavaFX. FXDiagram provides additional library classes to selectively add higher-level functionality trimmed for best user experience.

FXDiagram is almost entirely written in Xtend, which allows for very concise and readable code while still being fully compatible with Java.

FXDiagram integrates well with Eclipse. Adding a diagram view for an Xtext language is easy. Have a look at the example diagrams for Xtext's example languages domainmodel and fowlerdsl, as well as the generic class diagram browser for Java, Xtend and Xbase-languages. There is also a graphical tool for the analysis of Eclipse plug-in dependencies. To give them a try, point your Eclipse update manager to http://jankoehnlein.github.io/FXDiagram/p2/updates/latest

You can find more information on my blog.

Development

FXDiagram is currently being developed in Eclipse. The fastest way to get it running is to use Eclipse Oomph and the FXDiagram.setup. The configuration requires Java 8 SDK 40 early access build >= 5 (see below) and will install everything but Graphviz.

If you decide to setup the environment manually, you need

  • Java 8 SDK 40 early access build >=5 (https://jdk8.java.net/download.html)
  • Eclipse Luna 4.4
    • Xtext SDK 2.7
    • e(fx)clipse 1.1

You need some plug-ins from the KIELER project. Use the kielerProjectSet.psf to get them. Unfortunately, KIELER does not run without Equinox. If you want to run the standalone demo you have to apply the patch kieler.patch for some quick workarounds.

Graphviz must be installed on your system.
